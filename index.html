WCAG 2.2 Accessibility Validator

Analyse your Figma designs or live websites against WCAG 2.2 — get specific failures, reasons, and actionable fix suggestions.

How to get your free Figma Personal Access Token:
  1. Open Figma → click your avatar → Settings
  2. Scroll to Security → Personal Access Tokens
  3. Click "Add new token", give it any name, copy the token
  4. Paste it above — saved only in your browser's localStorage

Getting the correct share link: In Figma, open the design file → click Share (top-right) → Copy link. The URL should look like figma.com/design/… or figma.com/file/…

⚠️ Do not use a figma.com/site/… URL — that is a published Figma Sites page, not the design source file.
What gets checked: image alt text, form labels, heading structure, link text, page title & language, button names, duplicate IDs, iframe titles, autoplay media, table headers, and more — all against WCAG 2.2.

✅ Works well with: student projects, small business sites, university pages (e.g. library.westernsydney.edu.au), and any publicly-deployed site without bot protection.

⚠️ Will not work with: BBC, CNN, NYT, social media, and any site protected by Cloudflare or bot-detection — these services actively block automated proxy requests. Use the axe DevTools browser extension for those sites instead.

ℹ️ Other limitations: JavaScript-rendered content (React/Vue/Angular SPAs) and CSS-only colour contrast cannot be checked without a real browser render.

How to Get Your Figma Personal Access Token

Step-by-step guide for DESN2005 students — Western Sydney University

1
Open Figma and sign in
Go to figma.com and sign in to your account. You need the web app or desktop app — both work.
2
Click your profile avatar
In the top-left corner of the Figma home screen, click your profile picture or initials to open the account menu.
3
Open Settings
Select Settings from the dropdown. This opens your account settings page.
Click your avatar (top-left) → select Settings from the menu
4
Go to the Security tab
In the settings panel, click the Security tab in the left sidebar (or scroll down to find the Security section).
5
Create a new Personal Access Token
Find the Personal Access Tokens section and click "Add new token". Give it a name such as WCAG-validate, set expiration to 90 days, then click Generate.
Name the token (e.g. WCAG-validate) and set expiry to 90 days
6
Set token scopes (newer Figma accounts)
If Figma asks for permission scopes, find "File content" and set it to Read. This allows the validator to read your design data.
⚠️ If you skip this step the token will still be created but the validator will receive a 403 Access Denied error when it tries to read your file.
7
Copy the token immediately
Click "Generate token". Figma will show the token string only once — copy it right away. If you close the dialog without copying, you will need to generate a new one.
🔒 Your token is stored only in this browser's localStorage. It never leaves your device except to authenticate with Figma's API.
8
Paste it into the validator
Return to this tool, paste the token into the Personal Access Token field, and click Analyze Design.
